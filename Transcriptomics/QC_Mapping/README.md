### Quality control and filtering
The initial QC analysis of raw data was performed using FastQC v0.12.1: [fastqc.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/fastqc.sh). MultiQC v1.0 was used to create a report: [multiqc.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/multiqc.sh). After inspecting the results, raw reads were trimmed with Trimmomatic v0.39 using a custom adaptor [file](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/Sequencing_adaptors.fasta): [trimmomatic.multi.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/trimmomatic.multi.sh). The resulting read quality was accessed again with FastQC and summarized with MultiQC. The performance of rRNA depletion was tested with RiboDetector v0.3.1 tool with [RiboDetector v0.3.1](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/ribodetector.sh) and summarized with MultiQC and custom script [calc.perc.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/calc.perc.sh). Samples were renamed using [rename.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/rename.sh) script with [ids.csv](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/ids.csv). 

### Mapping
Filtered reads were aligned to *O. patagonica* genome [GCF_052425735.1](https://www.ncbi.nlm.nih.gov/datasets/genome/GCF_052425735.1/) using STAR v2.7.10b mapper. Firstly, genome index was created using genome fasta file and gff3 [annotation](https://osf.io/f7sc5) file: [index.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/index.sh) and reads were aligned to the genome with `--quantMode GeneCounts` option allowing to count the number reads per gene while mapping: [mapping.basic.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/mapping.basic.sh). The 3' coverage was accessed with RSeQC v.2.6.4 `geneBody_coverage.py` function ([rseqc.sh](https://github.com/talimass/Ocullina_Translocation/blob/main/Transcriptomics/QC_Mapping/rseqc.sh)).
